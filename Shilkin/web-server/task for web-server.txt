Написать на Java HTTP сервер, который должен уметь	Комментарий
1. Выводить список файлов в папке	"Папка из которой выводятся файлы задается в конфигурационном файле. 
Например, если в конфигурации оуказана папка C:\webserver.
то запрос GET / выведет все файлы и папки из c:\webserver. 
Запрос GET /mysite выведет все файлы и папки из c:\webserver\mysite (при условии, что такая папка существует)
Формат списка файлов - произвольный(например, HTML или JSON), но обязательно должен включать имя. "
2. Возвращать содержимое файла 	запрос GET /mysite/index.html выведет  содержимое файла c:/webserver/mysite/index.html
3. Создавать новый файл используя POST запрос	Запрос POST /mysite/new-file.txt создаст новый файл с содержимым, которые было передано в  теле запроса
4. Удалять существующий файл используя DELETE запрос	Запрос DELETE /mysite/new-file.txt удалит файл и вернет ответ 200 ОК, если файл есть и был удален. Или 404, если файла нет
	
Дополнительные требования	
0. Для реализации задания нужно использовать ServerSocket, без сервлетов.	
1. Для GET запроса должны поддерживаться клиентские заголовки, отвечающие за кэширование:
- If-None-Match
- If-Modified-Since	
2. Для поддерки кэширования со стороны сервера, ответ  на любой GET запрос должен включать загловки
- Etag (можно рассчитывать, как md5 от даты последнего изменения)
- Last-Modified  (просто дата последнего изменения файла)
"	
3. Добавление Basic авторизации. Имя и пароль задаются в конфигурации
