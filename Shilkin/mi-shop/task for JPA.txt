Сделайте REST интерфейс для управления интернет магазином:

- Есть сущности Товар: с полями Название, Категория, Цена, Остаток
- Есть сущность категория: Название, родительская категория
- Есть сущность заказ: Товар, количество, итоговая цена. Товаров может быть несколько, у каждого - свое количество.

Нужно создать eндпоинты для следующего:

1. Создание новой категории. Обязательно поле - название.

2. Создание нового товара - обязательные поля Название, категория, цена, остаток положительный.

3. Создание заказа.

4. Получение товара по названию

5. Получение товара по категории. Должны быть вывденеы товары так из подкатегорий.

Например, есть категория Продукты, у нее дочернаяя категория "Фрукты".

Есть Апельсины (Фрукты), Хлеб (Продукты). При запросе товаров из категории "продукты" должны быть выведены и
апельсины, и хлеб.

Для хранения товаров исползуйте БД, JРА и Spring Data
